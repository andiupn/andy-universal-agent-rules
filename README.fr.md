## 💡 Le problème : pourquoi les modèles d'IA intelligents écrivent du mauvais code
Les modèles d'IA avancés (tels que GPT-4o, Claude 3.5 Sonnet et Gemini Pro) ont une énorme capacité de raisonnement. Mais sans directives architecturales strictes, sans sandboxing clair des dossiers et sans règles de formatage unifiées, ils écrivent du code non structuré, ignorent vos conventions existantes et répètent les mêmes erreurs au fil des sessions.

---

## ⚡ La solution : la gouvernance de l'IA de référence

### 1. 📜 Règles universelles multi-éditeurs
Injecte des instructions d'IA hautes performances (équivalent `AGENTS.md` et `.cursorrules`) compatibles avec tous les principaux éditeurs d'IA : **Cursor, Windsurf, Claude Code, GitHub Copilot et Kiro**. Un ensemble de règles pour les gouverner tous.

### 🧠 2. Conservateurs de mémoire autonomes
Inclut des scripts Python stdlib natifs qui automatisent l’accumulation de connaissances sur les projets locaux. Si l'IA résout un bug ou prend une décision architecturale une fois, elle l'écrit dans `.agent/memory/` et ne l'oublie jamais.

### ⚡ Installateur sans dépendance
Non `npm install`, non `pip install`. S'installe en 1 seconde via un simple one-liner natif curl/Powershell, en téléchargeant uniquement les scripts Python de la bibliothèque standard et des flux de travail de démarque propres.

---

## 📂 Carte du répertoire visuel

```
your-project/
  .agent/
    scripts/           # Zero-dependency Python tools (Search, Save, Validate)
    workflows/         # Step-by-step markdown workflows (Maintenance, Save)
  AGENTS.md            # The master AI system instructions (entry point)
  install.sh           # Native Linux/macOS installer
  install.ps1          # Native Windows PowerShell installer
```

---

## 🚀 Installation (1 seconde)

### Windows (PowerShell) :

```powershell
iwr -useb https://raw.githubusercontent.com/andiupn/andy-universal-agent-rules/main/install.ps1 | iex
```

### Linux/MacOS (Bash) :

```bash
curl -fsSL https://raw.githubusercontent.com/andiupn/andy-universal-agent-rules/main/install.sh | bash
```

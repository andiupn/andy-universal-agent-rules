## 💡 Problema: de ce modelele inteligente de inteligență artificială scriu coduri greșite
Modelele avansate AI (cum ar fi GPT-4o, Claude 3.5 Sonnet și Gemini Pro) au o capacitate de raționament masivă. Dar fără linii directoare arhitecturale stricte, sandbox clar pentru foldere și reguli de formatare unificate, ei scriu cod nestructurat, ignoră convențiile existente și repetă aceleași greșeli în sesiuni.

---

## ⚡ Soluția: Guvernarea AI standard de aur

### 1. 📜 Reguli Universal Cross-Editor
Injectează instrucțiuni AI de înaltă performanță (echivalentul `AGENTS.md` și `.cursorrules`) compatibile cu toate editoarele AI majore—**Cursor, Windsurf, Claude Code, GitHub Copilot și Kiro**. Un set de reguli pentru a le guverna pe toate.

### 🧠 2. Curatori de memorie autonomi
Include scripturi native Python stdlib care automatizează acumularea locală de cunoștințe de proiect. Dacă AI rezolvă o eroare sau ia o decizie arhitecturală o dată, o scrie în `.agent/memory/` și nu o uită niciodată.

### ⚡ Instalator cu dependență zero
Nu `npm install`, nu `pip install`. Se instalează în 1 secundă printr-un simplu curl/Powershell one-liner nativ, descarcând numai scripturi Python din bibliotecă standard și fluxuri de lucru curate de reducere.

## 📂 Harta directorului vizual

```
your-project/
  .agent/
    scripts/           # Zero-dependency Python tools (Search, Save, Validate)
    workflows/         # Step-by-step markdown workflows (Maintenance, Save)
  AGENTS.md            # The master AI system instructions (entry point)
  install.sh           # Native Linux/macOS installer
  install.ps1          # Native Windows PowerShell installer
```

---

## 🚀 Instalare (1 secundă)

### Windows (PowerShell):

```powershell
iwr -useb https://raw.githubusercontent.com/andiupn/andy-universal-agent-rules/main/install.ps1 | iex
```

### Linux / macOS (Bash):

```bash
curl -fsSL https://raw.githubusercontent.com/andiupn/andy-universal-agent-rules/main/install.sh | bash
```

## 💡 Il problema: perché i modelli di intelligenza artificiale intelligente scrivono codici errati
I modelli di intelligenza artificiale avanzati (come GPT-4o, Claude 3.5 Sonnet e Gemini Pro) hanno un'enorme capacità di ragionamento. Ma senza rigide linee guida architettoniche, un chiaro sandboxing delle cartelle e regole di formattazione unificate, scrivono codice non strutturato, ignorano le convenzioni esistenti e ripetono gli stessi errori tra le sessioni.

---

## ⚡ La soluzione: la governance dell'IA Gold Standard

### 1. 📜 Regole universali per cross editor
Inserisce istruzioni IA ad alte prestazioni (equivalenti a `AGENTS.md` e `.cursorrules`) compatibili con tutti i principali editor IA:**Cursor, Windsurf, Claude Code, GitHub Copilot e Kiro**. Un insieme di regole per domarli tutti.

### 🧠 2. Curatori della memoria autonomi
Include script stdlib nativi di Python che automatizzano l'accumulo di conoscenza del progetto locale. Se l'IA risolve un bug o prende una decisione architetturale una volta, lo scrive su `.agent/memory/` e non lo dimentica mai.

### ⚡ Programma di installazione a dipendenza zero
No `npm install`, no `pip install`. Si installa in 1 secondo tramite un semplice one-liner nativo curl/Powershell, scaricando solo script Python della libreria standard e flussi di lavoro di markdown puliti.

## 📂 Mappa delle directory visive

```
your-project/

    scripts/           # Zero-dependency Python tools (Search, Save, Validate)
    workflows/         # Step-by-step markdown workflows (Maintenance, Save)
  AGENTS.md            # The master AI system instructions (entry point)
  install.sh           # Native Linux/macOS installer
  install.ps1          # Native Windows PowerShell installer
```

## 🚀 Installazione (1 secondo)

### Windows (PowerShell):

```powershell
iwr -useb https://raw.githubusercontent.com/andiupn/andy-universal-agent-rules/main/install.ps1 | iex
```

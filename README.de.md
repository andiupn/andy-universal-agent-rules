## 💡 Das Problem: Warum intelligente KI-Modelle schlechten Code schreiben
Fortgeschrittene KI-Modelle (wie GPT-4o, Claude 3.5 Sonnet und Gemini Pro) verfügen über eine enorme Argumentationskapazität. Aber ohne strenge Architekturrichtlinien, klares Ordner-Sandboxing und einheitliche Formatierungsregeln schreiben sie unstrukturierten Code, ignorieren Ihre bestehenden Konventionen und wiederholen dieselben Fehler sitzungsübergreifend.

---

## ⚡ Die Lösung: Die Gold Standard AI Governance

### 1. 📜 Universelle Cross-Editor-Regeln
Fügt leistungsstarke KI-Anweisungen ein (Äquivalent zu `AGENTS.md` und `.cursorrules`), die mit allen wichtigen KI-Editoren kompatibel sind – **Cursor, Windsurf, Claude Code, GitHub Copilot und Kiro**. Ein Regelwerk, um sie alle zu beherrschen.

### 🧠 2. Eigenständige Speicherkuratoren
Enthält native Python-Stdlib-Skripte, die die Ansammlung von lokalem Projektwissen automatisieren. Wenn die KI einmal einen Fehler behebt oder eine architektonische Entscheidung trifft, schreibt sie diese in `.agent/memory/` und vergisst sie nie.

### ⚡ Zero-Dependency-Installer
Nein `npm install`, nein `pip install`. Lässt sich in 1 Sekunde über einen einfachen nativen Curl/Powershell-Einzeiler installieren, wobei nur Standardbibliotheks-Python-Skripts und saubere Markdown-Workflows heruntergeladen werden.

## 📂 Visuelle Verzeichniskarte

```
your-project/
  .agent/
    scripts/           # Zero-dependency Python tools (Search, Save, Validate)
    workflows/         # Step-by-step markdown workflows (Maintenance, Save)
  AGENTS.md            # The master AI system instructions (entry point)
  install.sh           # Native Linux/macOS installer
  install.ps1          # Native Windows PowerShell installer
```

---

## 🚀 Installation (1 Sekunde)

### Windows (PowerShell):

```powershell
iwr -useb https://raw.githubusercontent.com/andiupn/andy-universal-agent-rules/main/install.ps1 | iex
```

### Linux / macOS (Bash):

```bash
curl -fsSL https://raw.githubusercontent.com/andiupn/andy-universal-agent-rules/main/install.sh | bash
```

## 💡 O problema: por que modelos inteligentes de IA escrevem códigos ruins
Modelos avançados de IA (como GPT-4o, Claude 3.5 Sonnet e Gemini Pro) têm enorme capacidade de raciocínio. Mas sem diretrizes arquitetônicas rígidas, sandboxing de pastas claro e regras de formatação unificadas, eles escrevem código não estruturado, ignoram as convenções existentes e repetem os mesmos erros nas sessões.

---

## ⚡ A solução: a governança de IA padrão ouro

### 1. 📜 Regras universais para editores cruzados
Injeta instruções de IA de alto desempenho (equivalentes a `AGENTS.md` e `.cursorrules`) compatíveis com todos os principais editores de IA—**Cursor, Windsurf, Claude Code, GitHub Copilot e Kiro**. Um conjunto de regras para governar todos eles.

### 🧠 2. Curadores de memória independentes
Inclui scripts stdlib nativos do Python que automatizam o acúmulo de conhecimento do projeto local. Se a IA resolver um bug ou tomar uma decisão arquitetônica uma vez, ela a grava em `.agent/memory/` e nunca a esquece.

### ⚡ Instalador de dependência zero
Não `npm install`, não `pip install`. Instala em 1 segundo por meio de um simples one-liner nativo curl/Powershell, baixando apenas scripts Python da biblioteca padrão e fluxos de trabalho de markdown limpos.

## 📂 Mapa do diretório visual

```
your-project/
  .agent/
    scripts/           # Zero-dependency Python tools (Search, Save, Validate)
    workflows/         # Step-by-step markdown workflows (Maintenance, Save)
  AGENTS.md            # The master AI system instructions (entry point)
  install.sh           # Native Linux/macOS installer
  install.ps1          # Native Windows PowerShell installer
```

---

## 🚀 Instalação (1 segundo)

### Windows (PowerShell):

```powershell
iwr -useb https://raw.githubusercontent.com/andiupn/andy-universal-agent-rules/main/install.ps1 | iex
```

###Linux/macOS (Bash):

```bash
curl -fsSL https://raw.githubusercontent.com/andiupn/andy-universal-agent-rules/main/install.sh | bash
```

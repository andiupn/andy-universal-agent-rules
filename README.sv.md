## 💡 Problemet: Varför smarta AI-modeller skriver dålig kod
Avancerade AI-modeller (som GPT-4o, Claude 3.5 Sonnet och Gemini Pro) har enorm resonemangskapacitet. Men utan strikta arkitektoniska riktlinjer, tydlig mappsandboxning och enhetliga formateringsregler skriver de ostrukturerad kod, ignorerar dina befintliga konventioner och upprepar samma misstag över sessioner.

---

## ⚡ Lösningen: The Gold Standard AI Governance

### 1. 📜 Universal Cross-Editor-regler
Injicerar högpresterande AI-instruktioner (`AGENTS.md` och `.cursorrules` motsvarande) som är kompatibla med alla större AI-redigerare—**Cursor, Windsurf, Claude Code, GitHub Copilot och Kiro**. En uppsättning regler för att styra dem alla.

### 🧠 2. Fristående minneskuratorer
Inkluderar inbyggda Python stdlib-skript som automatiserar ackumulering av lokal projektkunskap. Om AI:n löser en bugg eller fattar ett arkitektoniskt beslut en gång, skriver den det till `.agent/memory/` och glömmer det aldrig.

### ⚡ Zero-Dependency Installer
Nej `npm install`, nej `pip install`. Installeras på 1 sekund via en enkel inbyggd curl/Powershell one-liner, laddar bara ned standardbiblioteks Python-skript och rena markdown-arbetsflöden.

## 📂 Visuell katalogkarta

```
your-project/
  .agent/
    scripts/           # Zero-dependency Python tools (Search, Save, Validate)
    workflows/         # Step-by-step markdown workflows (Maintenance, Save)
  AGENTS.md            # The master AI system instructions (entry point)
  install.sh           # Native Linux/macOS installer
  install.ps1          # Native Windows PowerShell installer
```

---

## 🚀 Installation (1 sekund)

### Windows (PowerShell):

```powershell
iwr -useb https://raw.githubusercontent.com/andiupn/andy-universal-agent-rules/main/install.ps1 | iex
```

### Linux / macOS (Bash):

```bash
curl -fsSL https://raw.githubusercontent.com/andiupn/andy-universal-agent-rules/main/install.sh | bash
```

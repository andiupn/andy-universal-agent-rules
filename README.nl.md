## 💡 Het probleem: waarom slimme AI-modellen slechte code schrijven
Geavanceerde AI-modellen (zoals GPT-4o, Claude 3.5 Sonnet en Gemini Pro) hebben een enorm redeneervermogen. Maar zonder strikte architectonische richtlijnen, duidelijke map-sandboxing en uniforme opmaakregels schrijven ze ongestructureerde code, negeren ze uw bestaande conventies en herhalen ze dezelfde fouten tijdens sessies.

---

## ⚡ De oplossing: de gouden standaard AI-governance

### 1. 📜 Universele cross-editorregels
Injecteert hoogwaardige AI-instructies (equivalent van `AGENTS.md` en `.cursorrules`) die compatibel zijn met alle grote AI-editors:**Cursor, Windsurf, Claude Code, GitHub Copilot en Kiro**. Eén stel regels om ze allemaal te regeren.

### 🧠 2. Zelfstandige geheugencurators
Bevat native Python stdlib-scripts die de accumulatie van lokale projectkennis automatiseren. Als de AI een bug oplost of een architectonische beslissing neemt, schrijft hij deze naar `.agent/memory/` en vergeet deze nooit meer.

### ⚡ Zero-Dependency-installatieprogramma
Geen `npm install`, geen `pip install`. Kan in 1 seconde worden geïnstalleerd via een eenvoudige native curl/Powershell-oneliner, waarbij alleen Python-scripts uit de standaardbibliotheek worden gedownload en de markdown-workflows worden opgeschoond.

---

## 📂 Visuele mapkaart

```
your-project/
  .agent/
    scripts/           # Zero-dependency Python tools (Search, Save, Validate)
    workflows/         # Step-by-step markdown workflows (Maintenance, Save)
  AGENTS.md            # The master AI system instructions (entry point)
  install.sh           # Native Linux/macOS installer
  install.ps1          # Native Windows PowerShell installer
```

---

## 🚀 Installatie (1 seconde)

### Windows (PowerShell):

```powershell
iwr -useb https://raw.githubusercontent.com/andiupn/andy-universal-agent-rules/main/install.ps1 | iex
```

### Linux/macOS (Bash):

```bash
curl -fsSL https://raw.githubusercontent.com/andiupn/andy-universal-agent-rules/main/install.sh | bash
```
